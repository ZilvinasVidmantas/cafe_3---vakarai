Event listeners, classes and classlist

HTML yra tiesiog formatas kuris yra panaudojamas sudaryti document object model - DOM. Kiekvienas HTML elementas yra paverčiamas JavaScript objektu ir struktūrizuojamas medžio duomenų struktūra. Visa ši struktūra pasiekiama, naudojant JavaScript, kintamuoju - document

Kiekvieną kartą pakeitum document ar bet kurį kintamajį (HTML elemento objektą) jame, atsinaujina puslapio vaizdas. Šią dalį atlieka naršyklės "Render Engine". Kiekviena naršyklė turi savo 2 variklius: render engine ir JavaScript engine. Šie varikliai optimizuoja puslapio perpiešimo ir javascript veikimo darbus ir yra standartizuoti ECMA(European Computer Manufacturer Association) standarto.

Praktikoje, mums nei puslapio vaizdo perkūrimu, nei JavaScript kalbos plonybėmis rūpintis nereikia. Tai už mus padaro šie varikliai.

Dažniausiai JavaScript funkcionalumas aplikacijose yra įgalinamas naudojant įvykių klausiklius(Event Listeners). Tai daroma tokiu principu:
Įvykus įvykiui(paspausimas, užvedimas ir t.t.), vykdoma programuotojo sukurta funkcija.
Įvykių klausiklius galite uždėti ant bet kokio HTML elemento, o tiksliau jo HTML elemento JavaScript objekto.

Visualius pakitimus, dažniausiai įgaliname keisdami HTML elementų objektų CSS klases. Tai ir papraktikuosime šią pamoką.

Event listeners

Tai Node prototipo metodas, kuris reikalauja 2 parametrų

  1. Įvykis
  2. Funkcija, kurią vykdyti įvykus tam įvykiui

Įvykių gali būti įvairių. Bet koks HTML elementas gali reaguoti į paspaudiną, užvedimą. Įvesties gali reaguoti į naujo simbolio įvedima, sufokusavimą, atfokusavimą. Formos gali reaguoti į jų patvirtinimą, pats langas (ar bet koks scroll'inamas elementas) gali reaguoti į scroll'inimą.
